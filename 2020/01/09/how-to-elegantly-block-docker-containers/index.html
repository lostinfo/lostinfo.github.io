        如何优雅的阻塞 Docker 容器

        date 2020-01-09 tags docker、go、shell

        如果我们仅仅需要一个代码的运行环境,比如我需要搭建go的运行环境,而不想把代码拷贝进去,那么就会遇到这个问题。因为容器是一个进程,而且需要这个进程在前台运行,阻塞住shell,否则启动以后,docker主线程就会直接退出。但是现在我们没有用业务代码go run x.go来阻塞住shell,那怎么办呢?

        # 一、业务场景

        如果我们仅仅需要一个代码的运行环境,比如我需要搭建go的运行环境,而不想把代码拷贝进去,那么就会遇到这个问题。因为容器是一个进程,而且需要这个进程在前台运行,阻塞住shell,否则启动以后,docker主线程就会直接退出。但是现在我们没有用业务代码go run x.go来阻塞住shell,那怎么办呢?

        # 二、优雅的阻塞

        • 方法一:CMD["tail","-f","/dev/null"],使用linux系统的tail命令,可以阻塞住shell,容器不会退出。

        • 方法二:如果安装的工具中有能够阻塞shell的命令,可以直接它的命令,比如我在容器中安装了ssh便于我外部xshell连接到容器,那么就可以使用:CMD["/usr/sbin/sshd","-D"]

        • 方法三:使用docker的-dti参数,主线程会阻塞住shell, 比如docker run -dti --name example image,docker会后台运行,并且不会退出,但不适用docker-compose。

        # 三、业务代码需要注意的问题

        • 业务代码不能以守护模式运行,比如我用php写了一个多进程的服务,支持守护模式运行CMD["php" ,"index.php","start", "-d"],那么在容器里面,会导致容器退出,无法阻塞住shell
